```
Distributed System
```

```
No Node Accepts Writes
      |
```

**Solution:**

```
Automatic Leader Election
```

## Why Leader Election is Needed

```
Leader Crashes
      |
Cluster Detects Failure
      |
New Leader Elected
      |
System Continues Working
```

---

## What is Leader Election?

Process by which nodes in a cluster agree on a single coordinator node without manual intervention.

---

## How It Works

```
Nodes Send Heartbeats
      |
Leader Stops Responding
      |
Nodes Detect Timeout
      |
Election Triggered
      |
Node With Best Criteria Wins
      |
New Leader Announced
```

---

## Election Algorithms

```
Bully Algorithm
Raft Consensus
Zookeeper / etcd Based
```

### Bully Algorithm

Node with highest ID becomes leader.

```
Nodes: 1, 2, 3, 4, 5
Leader (5) Crashes
      |
Node 4 Detects Failure
      |
Sends Election Message to Higher IDs
      |
No Response → Node 4 Becomes Leader
```

### Raft Consensus

```
States: Follower, Candidate, Leader
```

```
Follower Times Out
      |
Becomes Candidate
      |
Requests Votes From Others
      |
Majority Votes → Becomes Leader
```

Used by: etcd, Consul, CockroachDB.

### Zookeeper-Based Election

```
Nodes Create Sequential Znodes
      |
Lowest Sequence Number = Leader
      |
Others Watch Previous Node
```

Used by: Kafka (older versions), Hadoop.

## Split Brain Problem

Two nodes think they are both leader.

```
Network Partition
      |
Node A Thinks It's Leader
Node B Thinks It's Leader
      |
Conflicting Writes
```

**Prevention:** Quorum-based majority voting.

```
Total Nodes = 5
Majority Needed = 3
Only Side With Majority Can Elect Leader
```

## Interview Questions

**What triggers a leader election?**
Missed heartbeats or a timeout indicating the current leader is unreachable.

**What is split brain?**
A scenario where a network partition causes two nodes to independently believe they are the leader, risking conflicting writes.

**How does quorum prevent split brain?**
By requiring a majority of nodes to agree before electing a leader, only one partition can ever reach majority.

---

## Summary

- Leader election allows clusters to recover automatically from leader failure.
- Bully, Raft, and Zookeeper-based are common algorithms.
- Split brain is prevented using quorum/majority voting.
- Used heavily in Kafka, MongoDB, Redis, etcd.
